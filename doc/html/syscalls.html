Syscalls, as defined by minilib

Syscall wrappers, x64.

Mostly untested, and possibly the used structures and typedefs are only predeclared, not defined.

All macros are prepended with 'sys_' (e.g. sys_read: 'read' is given as function, you could enable via themacro #mini_read; and the pure syscall-macro is there as well; sys_read)

..The difference between the macro read, provided by minilib and sysread are the arguments.

Minilib defines 'int', 'POINTER*' 'int'; the (generated) sys_read you can see below. .. And then .. Well, there's a good reason to define POINTER* instead of char*, but that would be a lengthy discussion; so here's just the generic advice: use the macros, by minilib, when provided - Otherwise resort to the syscalls given below. ;) When you tested one of the syscalls given below, it would be great you'd add the working syscall to syscallstubs as 'DEF_syscall(name,argcount, args... ) and do a merge request..

errno is set to sysret, when compiled with the switch #mini_errno.

There is also the possibilty to invoke syscalls by the generic macros syscall1 - syscall6. (TODO: document/syscall0)
